When an 'r' or 'R' prefix is present, a character following a backslash is included in the string without change, and all backslashes are left in the string. For example, the string literal r"\n" consists of two characters: a backslash and a lowercase 'n'. String quotes can be escaped with a backslash, but the backslash remains in the string; for example, r"\"" is a valid string literal consisting of two characters: a backslash and a double quote; r"\" is not a valid string literal (even a raw string cannot end in an odd number of backslashes). Specifically, a raw string cannot end in a single backslash (since the backslash would escape the following quote character). Note also that a single backslash followed by a newline is interpreted as those two characters as part of the string, not as a line continuation.

2.Regular Expressions to Cucumber Expressions

As shown above, you can keep using regular expressions and you can leave them untouched unless you’ve been using them with transforms, or you’ve got nested capture groups.

Here are some tips about how to translate Regular Expressions to Cucumber Expressions:

Anchors

The gorgeous ^ and $ you love so much are no longer needed. Just get rid of them.

Capture Groups

All capture groups become Cucumber Expression output parameters. There are 4 built-in ones to choose from:

  • (\d+) becomes {int}
  • Whatever funky pattern you’ve been using for floats becomes {float}
  • (\w+) becomes {word} (a word without quotes)
  • "([^"]+)" becomes {string} (anything between single or double quotes)

自学Python书籍推荐

发表于 2018-05-08 | 更新于 2018-09-23 | 分类于 Tech

自学Python书籍推荐

  • Python Cookbook： recipe类书籍，丰富的实例讲解编程过程中会遇到的各种情况。类似课后习题了，当你走过了入门阶段，这本书就是你日常使用中经常可以参考的一本宝典。
  • Fluent Python： 这本书我分了两次读完，第一次读很多地方没有体会过真的看不太懂。等到做了一段时间的项目后再回头阅读，觉得书如其名，一口气读下来感觉就是流畅。这本书像个经验丰富的老大哥，虽然不会面面俱到，但是对于初级到深入理解语言特性过程中会遇到的各种问题都有非常深入的理解，例子恰到好处，没有晦涩难懂的语言，语言精炼。每章最后的Further Reading会列出很多拓展阅读材料，可以看出作者阅读面之广，功力之深厚。强烈推荐的一本进阶书籍。
  • Effective Python：Best practice类书籍，像个尖子生，你总是可以以他为榜样，从而帮助你避开许多坑。
  • Learning Python：这本书我并没有读完，实在是太长了…我认为这本书最大的优点，同时也是缺点就是，太面面俱到了。对于入门这本书实际上并不合适，很快就会陷入枯燥的语法讲解导致昏昏欲睡了。但是如果你想系统的学习某块内容，那也许这本书就是你最好的选择。硬要形容的话这本书可能就像个大学教授，资历很深，但你未必能从他那很快得到你想要的答案吧。

测试框架开发小结(Draft)

发表于 2018-05-08 | 更新于 2018-09-23 | 分类于 Tech

测试框架开发小结(Draft):

一个好的测试框架应该具备以下特征:

Basic:

  • 准确, 测试结果必须是可靠的, pass就是pass, fail不是因为runtime error;

  • 稳定, 单个测试的fail不影响其他case的执行, 但某个错误会导致后续执行无意义则应尽早退出;

  • 灵活, 测试执行的范围是可灵活配置的, 按特性, 按tag, retry功能; 测试数据的允许自定义, 允许用户执行测试前执行特定的脚本, 允许用户配置测试执行的环境变量(比如token, key, username, pwd等信息);

  • 易拓展, 用户如何新增自己的测试场景, 如何定义自己的特定配置;

  • 完整性, 下载安装一次到位, 不需要多次安装依赖包等操作;

    Nice to Have:

  • 并发执行, 测试执行可以再多个环境上并发执行;

  • 兼容性, 支持跨平台执行;

  • 有数据分析, 一个好的测试框架应该提供直观的测试数据分析功能, 包括测试通过率, 执行时间, fail track, sync with bug system, write result to db;

需要明确的地方:

明确测试流程, 测试依赖, 测试数据来源, 测试干扰(网络延迟)

实现层面的考虑:

  1. Exception Hierarchy设计, custom Exception, handle Exception, 避免不必要的catch Exception;
  2. logging系统设计, console输出, file输出，production环境和test环境有不同的debug level

  3. Test Object设计, feature, scenario, case

    • 抽象类Test Object, tag, dependency, configuration属性, run_test, setup, teardown方法;

    • feature包含scenario, scenario包含case;

  4. Report系统, 将测试数据结构, pass, fail, skip数据以html方式展示;
  5. DB系统, 测试结果回写到后台数据库;
  6. Unix和Windows平台兼容性, 分隔符, 换行符处理等;
  7. 多线程, 测试数据准备, 测试执行并发系统(队列系统);
  8. 任务分发系统, 测试分发到不同executor上执行;

Python层面:

  1. 谨慎的使用tuple作为返回值, 有些返回值使用字典更加灵活;
  2. 设计好目录层级, 参考cucumber的folder;
  3. 测试的执行可以用配置文件来管理, 提供默认配置和按指定配置执行;

版本质量控制

  1. 分支代码合入流程规范：需要提PR，PR合入需要review，每次合入都应该仔细review，增量测试报告应该附上，避免灾难性的合入。Master分支合入后视情况进行daily的test，Next分支合入需要提供测试报告，Master主分支定期从Next分支合入代码；
  2. 原则上，每个模块都有各自负责的人，如果修改涉及到其他人模块的话，应该要求owner审核；

感悟

  • 用户永远是懒惰的，需求是会无限膨胀的，不要盲目满足用户需求，要注意解决用户痛点；
  • 用户对于解决方案是没有概念的，直到你把方案放在他面前才行；
  • 代码要自测！代码合入要谨慎！Reivew不能马虎！
  • 注意停滞的需求！

2016年4月出差回来之后，由于意识到自己的体重在毕业以后从58一路狂奔到72，长时间的电脑办公滋生了颈椎病。

那段时间正好在备孕，看着手机上今昔照片的对比，暗暗下定决心要开始锻炼身体了。

关于锻炼方式，因为我从小没有运动天赋，球类运动需要特定场馆而且需要有个固定的同伴比较好。

而我对于去健身房锻炼一向持保留态度（主要是穷），我一直坚信只要有心，在哪里锻炼都是可以的，所以剩下的只有一个选择了——跑步。

优点显而易见：

  • 无门槛（谁还不会跑步啊？！适合我这种没啥运动天赋的人）；
  • 自由（一个人想跑就跑，不像球类运行需要互相协调时间和场地）；
  • 有氧运动（我不会游泳，所以减肥最好的有氧运动就变成跑步了）；
  • 经济（你唯一必须的投资就是一双跑鞋，一套速干衣）

缺点嘛：

一个是枯燥，我跑步的时候不听歌，一个人跑40多分钟，黑灯瞎火的也没有啥靓丽的风景，如果不找点事确实是比较枯燥的。所幸的是，跑步的过程中脑子特别清醒，我利用这段时间将自己从一天喧嚣的工作中解脱出来，彻底放松自己，用来思考一些问题。

第二个就是计划赶不上变化，户外跑是很看天气的，杭州的梅雨季节我曾经停了3个星期没有去跑步。所以要提前做好规划，为了达到一周锻炼3-4次的要求，我买了个瑜伽垫，下雨天没法锻炼的时间就用keep补运动量。

过程

由于常年没有运动，一开始我只跑3.5公里。

那段时间家庭出现了重大经济危机和矛盾，工作上总有处理不完的棘手问题等着你，于是跑步成了我一天唯一的期盼。在我不知道应该如何应对眼前的问题的时候，跑步成了我唯一的坚持。虽然每次跑完都很累，但是比跑步更痛苦的是眼前的一堆问题。跑步成了我能短暂逃离现实的一种放松方式，同时也帮我在生活工作中慢慢的重拾信心。通过跑步，我为自己的意志力和信心充值。那段时间我正好在看《富爸爸穷爸爸》和《自控力》这两本书，从中收获良多并且结合自身情况也有诸多思考。跑步过程有个著名的“假疲劳”现象，就是你的疲惫感更多的是大脑自我保护的一个反应，并非真实的身体极限。当我将注意力集中在如何达成目标，而不是未自己寻找失败的原因的时候，一切变得不是那么困难了。我经常告诉自己，跑过这座桥才能停下，过了这个路灯就要开始跑步，不能在终点前停下。当我习惯于按照自己设立的原则和目标行动时，我确实跑的更远。我当时给自己立的标准是每个月的单次跑步里程必须增加500米，就这样我从开始的3.5公里，到现在的稳定7公里用了半年多时间。

当然，我想对于初学者来说，绝大多数人不是死在最后的1公里，而可能是开始的那100米。

就我自身而言，我发现如果我没有完成目标，极少是由于体力不支没有跑完。而90%的情况下，我是死在第一步——换衣服。因为到家时间比较晚，我一般要在8点以后才会拥有可自由支配时间，由于多年沉溺游戏的恶习，我经常一屁股就坐到电脑前面开始所谓的“relax time”。等到8点30我的streaks应用提醒我该运动时，一想到跑步这个漫长枯燥的过程，人的惰性渐渐战胜了理性，然后就这样平白无故缺勤了。这种状态一直持续了很久，我从开始的每周4跑渐渐的变成每周2跑，打卡多次中断。不断的反思和摸索中，我终于找到一种解决办法，我不再思考跑完需要花费我45分钟的时间这件事，而是聚焦于迈出第一步。比如今天该跑步了，到点的时候我只想着先去换个衣服吧，换完衣服时就自然而然去热个身，等热身完了我就下楼了，然后到了户外，看着自己自身行头跟前面的热身，想着不跑都对不起自己啊，于是就这样上路了！

通过分解任务，我将跑步这个过程划分为：

更换衣服》热身》下楼》迈开步子》跑完既定公里数

如果你也有同样的问题，不妨试试先分解任务。简单的将问题归因于自身意志力不足实际上是思想上的懒惰!

后记:

-写这篇文章是在2017年4月, 当时每次最远距离大概在8公里左右, 配速在每公里6分15秒左右;

-2018年5月5日更新, 目前每周大概跑步三次, 累计一个月的跑量在100-120公里左右. 最远距离在15公里, 10km的配速在每公里5分15秒左右;

回首这一年来的进步, 主要有两点, 一是偶然报名小马拉松时正好进了一个马拉松群, 各路大神每次打卡都是10km以上的, 在这种氛围刺激下不自觉的也提高了对自己的要求; 二是想着既然花那么多时间跑步了, 为什么不多努力一点让自己跑的更好更远呢? 于是开始系统的学习跑步的知识, 现在跑步已经完全融入我的生活, 并且一个好习惯常常会带来连锁反应, 早睡早起, 看书跑步, 写作思考, 这些习惯也能养成!
