盘点下这两年读过的书

发表于 2018-05-12 | 分类于 Life | 阅读次数：

盘点下这两年读过的书(未完)

GTD

  1. Getting Things Done: The Art of Stress-Free Productivity:
    GTD开山鼻祖
  2. ThePomodoroTechnique(番茄工作法):
    其实是指南, 很快就能读完, 每天专注学习2小时, 我个人是用滴答清单做番茄钟的因为有统计每天收获的番茄数, 趋势等等, 还能对具体的任务预估番茄;
  3. 高效能人士的七个习惯
    以7天为周期的第四代时间管理方法;
  4. The Power of Habit
    带你透视习惯养成的四个要素Cue>Routine>Reward & Belief;
  5. 吃掉那只青蛙
    手册类书籍, 速读即可.
  6. 拖延心理学
    成长心态和固定心态以及任务分解深有感触, 这本书写的非常详实, 不容错过的一本好书!
  7. 自控力
    我最早读的一本GTD系列书籍, 较<拖延心理学>短一些, 很快就能读完.

学习方法类:

  1. 如何阅读一本书:
    很长的一本书, 但是强烈推荐.

    读书的速度应该: 快不至于影响理解, 慢不至于不值得. 不同的书有不同的阅读方法;

    对于小说等虚构类书籍, 应该欣赏作者凭借你的想象力与情绪为你创造的世界;

  2. 刻意练习
    这本书还蛮火的, 但是个人认为水分不少, 内容冗余, 核心内容其实很少, 速读即可.
  3. 高效15法则
    职场高效工作指南. 书中的建议个人觉得都挺实用的, 尤其是开会, 邮件以及magic morning, 蛮有收获的一本书
  4. 谷歌工作整理术
    论共享信息与信息检索的重要性, 豆瓣分有点低, 多半抱怨作者硬广告以及国内无法正常使用谷歌产品. 但这些都不影响我们取其精华, 然后本土化~
  5. 暗时间
    其实是作者的博客集, 佩服作者的深刻洞察力. 最大的收获是让我坚定了写博客这件事;
  6. A Mind for Numbers:
    作者小学到高中数理成绩一路垫底，因此热爱文学。职场生活，不得不学习大量新鲜知识，甚至是头疼的数学知识。放下工作，回到学校，竟然学成了工程学博士，后留任教授。在MOOC、Coursera上开创最受欢迎的学习课“learning how to learn”，数十万人报名。关于学习方法的各个方面都提到了, 克服拖延, 番茄工作法, 专注模式与发散思维, 记忆等;

经管类:

  1. 卓有成效的管理者:
    管理学必读经典.

    管理者的任务不是去改变人。管理者的任务，在于运用每一个人的才干.

  2. 重新定义公司
    看看天才云集的公司是如何运营的

  3. 影响力
    雷布斯推荐读物

  4. 穷爸爸富爸爸

    改变了我很多观念

  5. 经济学原理

    这本是经济学必读经典, 写的通俗易懂, 一口气看下来的, 过瘾!

  6. 斯坦福极简经济学

    通俗易懂的一本书, 如果觉得经济学原理太长, 可以读这一本薄一点的.

  7. 牛奶可乐经济学
    兴趣读物, 提供一种思路, 不过感觉有些例子是强行分析的, 并没有实际验证;

  8. 怪诞行为学
    行为经济学, 第一次的定价会影响一件物品的;

  9. 小狗钱钱
    寓言类童话, 大人小孩都能读, 关键要个孩子从小树立理财的观念

传记类

  2. 乔布斯传
  3. 腾讯传
  4. 学习之道

码农系列:

  1. How Google Test Software
  2. The Cucumber Book, 2nd Edition
    如果要学cucumber, 首推这本书, 概念结合各类具体项目, 非常有收获的书!
  3. The Ruby Programming Language
    这本Ruby是写的最系统的, 作者之一是Ruby的创始人
  4. Python Cookbook, 3rd Edition
    手册类书籍, 都是干货
  5. How Linux Works, 2nd Edition
  6. Effective Python
    Best Practice of Python
  7. Python3 Object-oriented Programming - Second Edition
    这本是写的比较好的关于设计模式在Python中的具体实践.
  8. The Linux Command Line
    这一本写的简明易懂, 日常使用Linux系统都推荐读一下
  9. Fluent Python
    对Python的语言特性进行了深入分析, 建议有一定基础后再读.
